## 0. Executive Brand Read

Clutchr is not a baseball content library. It is not a generic sports psychology app. It is not a therapy app with baseball words pasted on top. It is a baseball-first career-path operating system that helps athletes know what to do next: before the game, during pressure, after mistakes, after rough outings, in training, and across a season.

The brand must feel like a serious baseball performance system with enough personality to be memorable. The visual direction should be premium, dark, athletic, sharp, green-forward, and baseball-native. The personality should come through Coach Cap, CC's Take, badges, microcopy, completion moments, and game-day surfaces, not through childish decoration or random mascot spam.

The core brand promise:

> **Clutchr helps baseball players build the routines, decisions, reset skills, and role-specific confidence that show up when the game speeds up.**

The internal creative standard:

> **If it does not feel like something a real ballplayer, coach, or serious parent would respect, simplify it or cut it.**

## 1. Product Identity

### 1.1 What Clutchr Is

Clutchr is a mobile baseball development app that blends:

- mental performance
- routines
- baseball IQ
- role-specific decision-making
- mechanics cues
- pressure reps
- game-day tools
- postgame debriefs
- readiness and recovery
- personal cue memory
- career-path progression

The app should feel like it knows the athlete's role, current state, season phase, and next best rep.

### 1.2 What Clutchr Is Not

Clutchr is not:

- a course library
- a video course platform
- a static content browser
- a wellness app
- a therapy app
- a quote-card app
- a generic confidence app
- a children's mascot game
- a quiz engine
- a social media feed
- a one-position pitching academy
- a cluttered module dashboard

### 1.3 The Category

The category Clutchr should own:

> **Baseball career-path operating system.**

Secondary descriptions:

- baseball mental performance system
- athlete routine and decision engine
- game-day preparation and reset app
- role-aware baseball development companion

Avoid using:

- mindset course
- mental health app
- meditation app
- motivation app
- learning module platform

## 2. Brand Positioning

### 2.1 One-Line Positioning

**Clutchr is the baseball performance app that gives athletes their next best rep, built around role, routine, pressure, and game-day execution.**

### 2.2 Longer Positioning

Clutchr helps baseball players train the parts of the game that do not show up cleanly in a box score: how they prepare, reset, make decisions, handle pressure, communicate, recover, and carry the right cue into the next rep. It uses short, playable baseball sessions instead of long lectures, and routes athletes through a personalized career path based on role, season, state, and progress.

### 2.3 Brand Pillars

| Pillar | Meaning | Product Expression |
|---|---|---|
| Baseball-native | Every feature must sound and feel like baseball. | Role-specific cues, real counts, base states, pressure moments. |
| Next best rep | The app should always answer what to do next. | Continue Career, Game Mode, postgame loop, readiness checks. |
| Role-aware | Pitchers, hitters, catchers, infielders, outfielders, and baserunners need different language. | Role tags, route logic, role-specific UI copy. |
| Premium but practical | It should feel polished without getting complicated. | Dark UI, clean cards, short sessions, strong typography. |
| Calm under pressure | Clutchr teaches visible behaviors, not abstract hype. | Quick resets, routines, pressure reps, boss battles. |
| Personality with restraint | Coach Cap adds identity without turning the app childish. | CC's Take, subtle appearances, completion moments. |

## 3. Audience Definition

### 3.1 Primary Users

Baseball players ages roughly 10-24, especially:

- serious youth athletes
- travel ball players
- middle school and high school players
- college-aspiring players
- high school varsity players
- college players who want structured routines and mental reps

### 3.2 Secondary Users

- parents who want credible development support
- coaches who value process and role discipline
- private instructors who want athletes to carry cues between sessions
- teams that want shared language without building admin software

### 3.3 Audience Feel Test

The app must be:

- cool enough for a 12-year-old to open without embarrassment
- credible enough for a serious high school player
- clean enough for a parent to trust
- sharp enough for a coach to respect
- not so kiddish that athletes outgrow it
- not so sterile that it feels like school

## 5. Voice and Copy Rules

### 5.1 Core Voice Rules

Use language that is:

- baseball-native
- role-aware
- short
- cue-based
- practical
- clear
- pressure-usable
- specific to a moment

Avoid language that is:

- therapy-coded
- classroom-coded
- generic motivational
- overly academic
- overly cute
- over-explained
- filled with hype punctuation

### 5.2 Approved Voice Patterns

Use copy like:

- Attack the zone.
- One pitch.
- See it early.
- Finish the rep.
- Stay in rhythm.
- Reset the inning.
- Own your tempo.
- Keep the team calm.
- Read it. Commit.
- Ball first. Then decision.
- Do not donate the next rep to the last one.

### 5.3 Banned Copy Patterns

Avoid:

- Believe in yourself.
- You got this.
- Be your best self.
- Manifest your success.
- Mindfulness journey.
- Mental health check-in.
- Therapy session.
- Confidence booster.
- Course module.
- Quiz complete.
- Generic “Good job!”
- Let's gooooo!!!

### 5.4 Button Copy Rules

Buttons should use verbs and outcomes.

Preferred:

- Continue Career
- Start Reset
- Finish Rep
- Complete Lesson
- Save Routine
- Watch Clip
- Try Again
- Build Plan
- Open Playbook
- Start Game Prep
- Log Debrief

Avoid:

- OK
- Submit
- Proceed
- Yes/No
- Next, when a stronger action is available

### 5.5 “Lock In” Decision

The old “Lock In” direction should be used carefully. It can work as brand language, but it is not always clear enough as a final action.

Best completion-action candidates:

| Candidate | Best use |
|---|---|
| Finish Rep | Best brandable lesson-completion action. |
| Complete Lesson | Clearest and safest. |
| Finish Strong | Good for end-of-boss or end-of-sequence. |
| Close It Out | Strong baseball flavor for boss/completion moments. |
| Lock It In | Use sparingly for cue commitment, not every button. |

Recommendation:

- Use **Finish Rep** for branded lesson completion.
- Use **Complete Lesson** when clarity matters most.
- Use **Lock It In** only when the athlete is saving a cue, routine, or commitment.

## 6. Visual Identity Direction

### 6.1 Visual North Star

Clutchr should look like a premium baseball performance app, not a classroom app and not a childish gamified app.

Visual references by feeling, not copying:

- premium sports training apps
- modern baseball broadcast graphics
- dark-mode fitness apps
- performance dashboards
- collectible sports badges
- clean App Store screenshots
- subtle stadium lighting
- green signal energy

### 6.2 Core Visual Traits

The visual system should be:

- dark
- high-contrast
- clean
- sharp
- athletic
- card-based
- slightly gritty
- modern
- premium
- green-accented
- motion-light
- icon-forward but not icon-cluttered

### 6.3 Primary Color System

Recommended starting palette:

| Role | Color | Hex | Usage |
|---|---:|---|---|
| Primary Black | Almost black | `#050806` | App background, deep surfaces. |
| Charcoal | Dark surface | `#111612` | Cards, panels, modal surfaces. |
| Field Green | Brand green | `#23D160` | Primary CTA, progress, success, active state. |
| Electric Green | Bright accent | `#39FF88` | Glow, streak, reward pulses, highlights. |
| White | Clean text | `#F7FFF9` | Primary text. |
| Cool Gray | Support text | `#A8B3AA` | Secondary text, descriptions. |
| Deep Gray | Borders | `#242B26` | Card outlines, dividers. |

### 6.4 Accent Colors

Use accents sparingly.

| Accent | Hex | Best use |
|---|---:|---|
| Gold | `#F5C542` | Badges, premium, mastery. |
| Red | `#FF4D4D` | Caution, mistake audit, urgent Game Mode only. |
| Purple | `#8B5CF6` | Advanced Signal/premium content. |
| Ice Blue | `#63D7FF` | Recovery/readiness, calm metrics. |
| Rose Gold | `#D99A6C` | Higher prestige badges. |

### 6.5 Color Rules

- Green is the energy signal. Do not put it everywhere.
- Black and charcoal should dominate the app.
- White text should remain crisp and readable.
- Gold should feel earned, not decorative.
- Red should be functional, not dramatic.
- Purple should signal advanced/premium/Signal content.
- Gradients should be subtle.
- Avoid rainbow gamification.

## 7. Typography Direction

### 7.1 Typography Personality

Typography should feel athletic, strong, readable, and modern.

Recommended direction:

- strong condensed or semi-condensed display style for major headers
- clean sans-serif for body copy
- bold labels for role tags, XP, and rank
- large, readable text on lesson screens

### 7.2 Type Rules

| Surface | Style |
|---|---|
| Main headers | Bold, athletic, high contrast. |
| Page subtitles | Short, gray, practical. |
| Lesson prompt | Large and readable. |
| Button text | Bold, clear, action-first. |
| Role tags | Small caps or compact badges. |
| CC's Take | Slightly branded, but still readable. |

### 7.3 Text Density Rule

Mobile screens should use one main idea per view.

Bad:

> Long explanation, three subheadings, four buttons, five cards, and a mascot all fighting for attention.

Good:

> One baseball moment. One decision. One cue. One action.

## 8. Logo and App Icon Direction

### 8.1 Logo Goal

The Clutchr logo should feel like a serious sports brand, not a startup productivity tool and not clip-art baseball.

### 8.2 Best Logo Directions

Explore:

1. **Angular wordmark**  
   A strong CLUTCHR wordmark with athletic cuts and subtle home-plate geometry.

2. **C mark**  
   A bold stylized C that can stand alone as an app icon.

3. **Home plate mark**  
   A home plate shape that doubles as a progress/rank symbol.

4. **Seam-negative-space mark**  
   Subtle baseball seam geometry, not literal ball clip art.

5. **Coach Cap silhouette lockup**  
   Secondary brand mark only; do not make the mascot the only logo.

### 8.3 Logo Rules

- Must work at app-icon size.
- Must work in one color.
- Must work on dark and light backgrounds.
- Must not look like a youth team logo.
- Must not use generic crossed bats unless heavily abstracted.
- Must not rely on tiny details.

### 8.4 App Icon Recommendation

Best first icon direction:

> **A sharp home-plate-shaped C mark in black and green.**

Reason: it connects baseball, progress, app identity, and future badge language.

## 9. Coach Cap Character System

### 9.1 Role of Coach Cap

Coach Cap is the original in-app guide for Clutchr. He should add personality, trust, and recognizable brand memory.

He is not:

- a goofy mascot
- a comic relief character
- a cartoon narrator on every screen
- the main product
- a distraction from the athlete's path

He is:

- a guide
- a coach figure
- a brand anchor
- a signal for advice, correction, and celebration
- the voice behind CC's Take

### 9.2 Character Personality

Coach Cap should feel:

- calm
- confident
- sharp
- baseball-smart
- slightly mysterious
- supportive but not soft
- cool enough for older athletes
- friendly enough for younger athletes

### 9.3 Visual Direction

Coach Cap should be:

- simplified baseball silhouette
- cap-forward identity
- readable at small sizes
- dark body/base with green accent
- not overly detailed
- not childish
- not a bobblehead
- not a furry mascot

### 9.4 Required Pose Library

Create and maintain these reusable Coach Cap assets:

| Pose | Use |
|---|---|
| Neutral stance | Onboarding, empty states, general guidance. |
| Pointing / giving advice | CC's Take, coaching correction. |
| Calm focus | Quick Reset, slump, pressure prep. |
| Celebration | Lesson completion, badge unlock. |
| Thinking | Scenario Pick, Film Read, Opponent Intel. |
| Clipboard / scouting | Game Mode, scouting, Playbook. |
| Arms crossed | Profile, standards, progress review. |

### 9.5 Coach Cap Usage Rules

Use Coach Cap:

- after completion moments
- on CC's Take cards
- in onboarding lightly
- in empty states
- during quick tips
- in marketing assets
- in App Store screenshots

Do not use Coach Cap:

- on every lesson step
- as decoration with no purpose
- in serious recovery warnings
- in every header
- when screen clarity matters more

## 10. CC's Take System

### 10.1 What It Is

**CC's Take** is the branded coaching insight format inside Clutchr.

It is a short, practical, baseball-native coaching note that connects a lesson, mistake, routine, or next action to one useful cue.

### 10.2 Format

**CC's Take** should usually contain:

1. One short insight.
2. One baseball reason.
3. One next cue or action.

Example:

> **CC's Take**  
> You did not chase panic. Good. When the game speeds up, shrink it to one job: breath, target, attack.

### 10.3 Best Surfaces

Use CC's Take on:

- lesson completion screens
- boss debriefs
- Home after rough session
- postgame debrief summary
- Playbook cue save confirmation
- App Store screenshots
- social graphics

### 10.4 Rules

- Keep it short.
- Do not use generic praise.
- Name the baseball behavior.
- Give a next cue.
- Do not overuse it.
- Do not make CC's Take a long article format.

## 11. UI System Principles

### 11.1 Overall UI Rule

Clutchr should feel hidden but obvious: the athlete should not see the full complexity of the system, but the next action should be clear.

### 11.2 Main Navigation Philosophy

The app should center:

- Home
- Career
- Game Mode
- Resources
- Profile

Home remains the main command center. Career is the progression path. Game Mode is for baseball-day tools. Resources is the content/reference layer. Profile is identity, progress, and personal operating system.

### 11.3 Home Screen Rule

Home should not become a dashboard junk drawer.

Home should show:

1. brand header
2. dominant Continue Career CTA
3. today’s focus / next best rep reason
4. current progress signal
5. one contextual utility action
6. optional CC's Take or Coach Cap moment

Do not show:

- too many cards
- all worlds
- all resources
- generic content browsing
- multiple equal CTAs

### 11.4 Career Screen Rule

Career should feel like a climb, not a syllabus.

Best model:

- one active world
- one teaser world
- completed work collapsed
- state-triggered interruption when necessary
- premium Signal teaser rail if needed

Avoid:

- 20 locked worlds at once
- pillar shopping
- visible textbook chapters
- giant course catalog behavior

### 11.5 Game Mode Rule

Game Mode should answer:

> **What are you trying to do right now?**

Recommended categories:

- Get Ready
- Reset
- Refocus
- Recover
- Debrief
- Intel / Today’s Plan, later

Game Mode must be fast, practical, and baseball-day usable.

### 11.6 Resources Rule

Rename Locker to **Resources** unless there is a strong future reason not to.

Recommended top tabs:

| Tab | Purpose |
|---|---|
| Dugout | Hitting, offense, team, general baseball performance. |
| Bullpen | Pitching, catching, battery, arm care, throwing. |
| The Grind | strength, recovery, development habits, standards. |

Resources should feel organized and useful, not like a blog feed.

### 11.7 Profile Rule

Profile should become the athlete’s baseball identity and personal operating system.

It should eventually include:

- role
- season phase
- current build areas
- badges
- saved cues
- saved routines
- Playbook
- progress history
- postgame trends
- readiness/recovery signals

Profile should not become a noisy analytics dashboard too early.

## 12. Component Style Guide

### 12.1 Cards

Cards should be:

- rounded
- dark charcoal
- cleanly spaced
- strong title hierarchy
- subtle border or glow
- not over-shadowed
- not visually flat

Recommended card formula:

- small role/status tag
- strong title
- one practical subtitle
- clear action
- optional icon/Coach Cap accent

### 12.2 Buttons

Primary CTA:

- green fill
- black or near-black text if contrast works
- bold label
- large touch target
- subtle glow only on important moments

Secondary CTA:

- dark fill
- green border or gray border
- white text

Danger/warning CTA:

- red used sparingly
- clear language
- never dramatic

### 12.3 Tags and Pills

Pills should feel clean, not bubbly.

Use for:

- role
- time estimate
- XP
- difficulty
- state tag
- premium
- active category

Avoid overusing pills as decoration.

### 12.4 Progress Elements

Progress should feel earned and clean.

Use:

- progress bar
- badge ring
- milestone cards
- world completion strip
- subtle XP count-up

Avoid:

- excessive confetti
- coin overload
- childish reward explosions
- too many stats at once

## 13. Reward and Badge Identity

### 13.1 Reward Philosophy

Rewards should represent:

- earned confidence
- proof of work
- visible progress
- quiet momentum
- role-specific mastery
- baseball standards

Rewards should not feel like cheap dopamine.

### 13.2 Badge Visual Direction

Best progression symbol:

> **Home plate badge system.**

Reason: baseball-native, clean, scalable, and consistent with app icon/logo direction.

### 13.3 Badge Ladder

| Tier | Visual | Feel |
|---|---|---|
| Green Plate | green/black home plate | first proof of work |
| Sharp Plate | refined green plate with stronger edge | consistency |
| Steel Plate | brushed metal | serious reps |
| Gold Plate | gold accent | mastery |
| Rose Gold Plate | rare prestige | advanced consistency |
| Platinum Crest | elite badge | high-level completion |
| Clutchr Elite Emblem | custom championship crest | endgame status |

### 13.4 Reward Copy Rules

Good reward copy:

- Routine locked in.
- Command stronger.
- Reset skill sharper.
- Boss cleared.
- Team calm protected.
- Damage zone set.
- First move cleaner.
- Next rep ready.

Avoid:

- Amazing job!
- You are unstoppable!
- You earned coins!
- Confidence boosted!

## 19. Do / Don’t Summary

### 19.1 Do

- Make the next action obvious.
- Use dark premium design.
- Use green as a signal color.
- Build Coach Cap as a restrained guide.
- Use CC's Take as a branded advice system.
- Keep lessons and tools short.
- Make rewards feel earned.
- Use baseball counts, roles, base states, and game moments.
- Build around Home, Career, Game Mode, Resources, and Profile.
- Keep UI surfaces consistent.
- Make App Store assets polished and serious.

### 19.2 Do Not

- Turn Clutchr into a course catalog.
- Show too many worlds at once.
- Use childish cartoon energy.
- Put Coach Cap everywhere.
- Use generic sports motivation.
- Use therapy or wellness language.
- Add random tabs.
- Add social feeds in the core product.
- Overload Home.
- Make rewards feel like coins/candy.
- Use unsupported lesson variants in implementation prompts.
- Prioritize pretty visuals over app clarity.

## 20. Final Brand Test

Before shipping any design, copy, screenshot, or UI change, ask:

1. Does this feel baseball-native?
2. Does this help the athlete know what to do next?
3. Would a serious high school player respect this?
4. Would a parent trust this?
5. Would a coach understand this quickly?
6. Is Coach Cap helping or distracting?
7. Is the screen clean enough for mobile?
8. Is the copy short and actionable?
9. Does this avoid therapy/classroom/generic motivation language?
10. Does this make Clutchr feel more premium and more useful?

If the answer is no, revise before coding or generating more assets.
